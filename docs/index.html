PulseDeck

A lightweight Python desktop monitor that watches your CPU, RAM, disk, GPU, and processes in real time — detects load spikes, surfaces the culprits, and lets you kill them safely. Without weighing your machine down.

מוניטור שולחני קליל ב-Python שעוקב אחר ה-CPU, RAM, דיסק, GPU והתהליכים שלך בזמן אמת — מזהה עומסים, חושף את הגורמים האשמים, ומאפשר לסגור אותם בבטחה. בלי להכביד על המחשב.

Python 3.10+ Windows · Linux · macOS MIT License ~80 MB RAM < 1% CPU

Why it exists

Windows Task Manager is great when you're already in trouble. PulseDeck is for the 30 seconds before that — when your machine starts to stutter and you want to know, without clicking, what's actually eating the CPU right now.

Three promises.
  1. Real-time data — second-by-second sampling, logged to CSV for analysis.
  2. Spike alerts — gentle toast in the bottom-right when load jumps. Click for the top processes responsible.
  3. Low overhead — adaptive sampling, batched system calls, incremental file reads. The monitor never becomes the problem it's monitoring.

Features

📊 Live monitoring

CPU · RAM · Disk · GPU · VRAM · Network · Temperature · Battery · Uptime. All sampled every second, written to CSV.

📈 Interactive chart

Embedded matplotlib live chart with time-window selector (5 min / 15 min / 1 h / 6 h / 24 h / all). Pan, zoom, save as PNG.

🔔 Spike alerts

Bottom-right toast notification, auto-dismiss in 8 s. Click to see top 5 CPU + top 5 RAM processes with activity indicators.

✕ Safe Kill

Kill a runaway process from the alert window. Always asks confirmation. System-critical processes (svchost, csrss, …) are protected.

🧹 Process Janitor

Detects accumulated conhost.exe zombie groups from dev CLIs. One-click cleanup, audited log, never automatic.

📁 Smart history

Weekly CSV rotation, per-day Markdown reports for spike events, configurable retention (default 12 weeks).

🎛 Three UI modes

Floating dock above the taskbar · system-tray icon · plain console. Mix and match. Dock supports drag, resize, pin.

⚙ Persistent config

Preferences saved to config.json. CLI flags override. --save-config writes current run back.

Quick start

git clone https://github.com/<your-user>/pulsedeck.git
cd pulsedeck
pip install -r requirements.txt

# Run with dock + history + tray
python monitor.py --dock --history --tray

# Install a desktop shortcut (Windows .lnk / Linux .desktop)
python monitor.py --install-shortcut

On first run, config.json is created from defaults and lives next to monitor.py. To start at every boot on Windows, drop a shortcut to Start-Monitor-Hidden.vbs into shell:startup.

How alerts work

The sampler compares each new metric snapshot with the previous one. A spike is logged when:

  • CPU rises 12 percentage points (or falls — both directions trigger).
  • RAM rises 6 percentage points.
  • CPU exceeds 88% sustained.
  • RAM exceeds 92% with at least 2 pp uptick.

When triggered (and the 5-minute cooldown is clear), a small toast slides into the bottom-right corner. Left-click opens the full alert window — top 5 CPU and top 5 RAM processes side by side, with activity badges (green = active now, amber = active recently, gray = background, purple = protected), uptime, and a flat red Kill button. Right-click the toast for snooze options (15 / 30 / 60 minutes).

Killing is always manual. Even with the "Clean all" button on the Janitor panel, every action goes through a confirmation dialog. The monitor will never terminate a process on its own.

Performance — how it stays cheap

SubsystemOptimization
Process sampler Adaptive rate — 2 s during the 30 s window after an alert; 5 s when idle. Resumes the fast rate immediately when a new alert fires.
Live chart Incremental CSV reader — only the appended tail since the previous refresh is parsed. 86× faster than a full re-read of a 1.5 MB metrics file.
Janitor Parent-name cache — one psutil.Process() call per unique parent PID, not per zombie.
Dock Layout pass only on change — no per-tick window repositioning. Topmost re-assertion throttled to every 5 s.
Logging Cooldown-suppressed events demoted from INFO to DEBUG; spike-oscillation deduplicated within a 30 s window.

Threading: one daemon thread per long-lived service (process sampler, janitor scanner). All UI updates funnelled to the Tk main thread via root.after(). No multiprocessing, no asyncio — small dependency surface, easy to read.

Hardware & OS compatibility

PulseDeck is designed to run on any machine — Intel, AMD, ARM, Apple Silicon, etc. Core monitoring works everywhere via psutil. Sensor-level metrics (GPU, CPU temperature) are read with graceful fallback: if the hardware doesn't expose a sensor, the value simply shows — and the app keeps running.

למה זה קיים

ה-Task Manager של Windows מצוין כשאתה כבר בצרות. PulseDeck נועד ל-30 השניות שלפני — כשהמחשב מתחיל לגמגם ואתה רוצה לדעת, בלי ללחוץ, מה בדיוק אוכל את ה-CPU כרגע.

שלוש הבטחות.
  1. נתונים בזמן אמת — דגימה בכל שנייה, רישום ל-CSV לניתוח.
  2. התראות עומס — toast עדין בפינה הימנית-תחתונה כשהעומס קופץ. לחיצה — והנה רשימת התהליכים האחראיים.
  3. טביעת רגל נמוכה — דגימה אדפטיבית, קריאות מערכת מקובצות, קריאות קובץ אינקרמנטליות. המוניטור אף פעם לא הופך לבעיה שהוא מנטר.

איך התראות עובדות

הסמפלר משווה כל snapshot חדש עם הקודם. spike נרשם כאשר:

  • CPU עולה ב-12 נקודות אחוז (או יורד — שני הכיוונים מפעילים).
  • RAM עולה ב-6 נקודות אחוז.
  • CPU מעל 88% באופן מתמשך.
  • RAM מעל 92% עם עלייה של 2 נקודות לפחות.

כאשר מופעל (וה-cooldown של 5 דקות פנוי), toast קטן מחליק לפינה הימנית-תחתונה. קליק שמאלי פותח את חלון ההתראה המלא — Top 5 CPU ו-Top 5 RAM זה לצד זה, עם תגי פעילות (ירוק = פעיל עכשיו, ענבר = פעיל לאחרונה, אפור = ברקע, סגול = מוגן), זמן ריצה, וכפתור Kill אדום שטוח. קליק ימני על ה-toast — אפשרויות snooze (15 / 30 / 60 דקות).

הריגה היא תמיד ידנית. גם עם הכפתור "נקה הכל" בפאנל ה-Janitor, כל פעולה עוברת דיאלוג אישור. המוניטור אף פעם לא יסיים תהליך לבדו.

ביצועים — איך זה נשאר קל

תת-מערכתאופטימיזציה
סמפלר תהליכים קצב אדפטיבי — 2 שניות בחלון של 30 שניות אחרי alert; 5 שניות במצב idle. חוזר לקצב המהיר מיד כשמתפוצץ alert חדש.
גרף חי קריאת CSV אינקרמנטלית — רק ה-tail שנוסף מאז ה-refresh הקודם נקרא. פי 86 מהיר יותר מקריאה מלאה של קובץ 1.5 MB.
Janitor מטמון parent-name — קריאה אחת ל-psutil.Process() לכל PID-הורה ייחודי, לא לכל זומבי.
דוק פריסה רק בשינוי — אין re-position בכל tick. אכיפה של topmost רק כל 5 שניות.
Logging אירועי cooldown-suppressed הורדו מ-INFO ל-DEBUG; spike-oscillation מבוטל בתוך חלון של 30 שניות.
